# Archivist-Library — Аудит проекта

## Что это за сайт

Archivist-Library — русскоязычный справочник и набор инструментов для мобильной игры Mutants Genetic Gladiators. Сайт живёт по адресу archivist-library.com, а статика (картинки, шрифты) раздаётся через CDN на cdn.archivist-library.com.

Проект объединяет вики-раздел с каталогом мутантов, пять симуляторов игровых механик, два калькулятора, базу материалов и несколько справочных страниц. Все вычисления шансов воспроизводят реальную игровую логику.

---

## Технологический стек

**Ядро:** Astro 5 в режиме SSR (output: 'server') с деплоем на Vercel через @astrojs/vercel. TypeScript в strict-режиме.

**Интерактивность:** Svelte 5 — все динамические элементы (симуляторы, браузер мутантов, калькуляторы, лидерборд) реализованы как Svelte-компоненты с гидратацией client:load.

**Стили:** Tailwind CSS 4 подключается как Vite-плагин. Глобальные стили живут в BaseLayout.astro (инлайн <style>) и в src/styles/global.css. Тёмная тема с фоновым изображением, blur-эффектами и пятью брейкпоинтами (мобилка, планшет, десктоп, 2K, 4K с автоматическим масштабированием шрифта).

**Данные:** Игровые данные хранятся в src/data/ в форматах JSON (мутанты, бинго, гача, материалы), XLSX (тир-лист, топ эво), TXT-файлы (локализация, base-данные, рецепты крафта в формате, похожем на XML). Google Sheets используется для живого лидерборда эво (парсинг CSV, кэш 5 минут). TXT-файлы импортируются как raw-строки через vite-plugin-raw, XML парсится fast-xml-parser.

**Дополнительно:** Sharp для оптимизации изображений, xlsx для чтения Excel-файлов на этапе сборки, dom-to-image-more для конвертации DOM в картинки.

## Архитектура и структура

### Как организован рендеринг

Большинство страниц — статические (prerender = true). Некоторые работают в серверном режиме: /top-evo грузит данные из Google Sheets при каждом запросе, /api/telegram-webhook обрабатывает POST-запросы от Telegram-бота. Маршруты реактора (/simulators/reactor/[gachaId]) генерируются через getStaticPaths() — один URL на каждый генератор.

### Компонентный подход

Astro-компоненты (.astro) отвечают за структуру страниц, передачу данных и статичную разметку. Svelte-компоненты (.svelte) — за всю интерактивность: симуляторы прокруток, браузер мутантов с поиском и модалкой, калькуляторы с реактивными полями ввода, leaderboard с анимациями.

### Лейаут

Единственный лейаут — BaseLayout.astro. Он задаёт: sticky-хедер с десктопной навигацией и dropdown-меню, мобильный сайдбар с overlay, footer, OG-теги, прелоад шрифта, анализ Vercel. Весь кастомный CSS для хедера/футера/меню инлайновый — компонент Self-contained.

### Стилизация

Два уровня: кастомный CSS (переменные для цветов, радиусов, отступов; стили для header/footer/mobile-nav) и Tailwind-утилиты в компонентах. Адаптивность покрывает экраны от 320px до 4K, причём на 2K+ автоматически увеличивается базовый размер шрифта.

## Карта сайта

**Главная навигация (хедер):** Мутанты | Калькуляторы ▾ | Материалы | Симуляторы | Скрещивания | Бинго | Тир-лист | Разработчики | Топ по эво

### Каталог мутантов
- /mutants — общий браузер с фильтрами по гену, типу, бинго. Поиск с нормализацией (латиница→кириллица, убирает спецсимволы, апострофы). Модальное окно с деталями мутанта.
- /mutants/bronze, silver, gold, platinum, normal — страницы по тирам редкости.

### Калькуляторы
- /evolution/evotech-calculator — расчёт затрат ресурсов для эволюции мутанта.
- /simulators/stats — точный расчёт финальных статов (HP, ATK1, ATK2, способность, скорость, серебро) с учётом уровня и множителя звёзд.

### Симуляторы
- /simulators — хаб со ссылками на все симуляторы.
- /simulators/breeding — предсказание потомства от скрещивания двух мутантов. Показывает генные комбинации, вероятности из XML-настроек, секретные рецепты и обратный поиск родителей.
- /simulators/reactor — выбор одного из ~20 генераторов (Вестерн, Япония, Шахматы и т.д.), каждый со своей обложкой.
- /simulators/reactor/[gachaId] — симуляция прокруток конкретного генератора с шансами из gacha.json.
- /simulators/roulette/cash — Cash Machine: прокрутка на золото/серебро с джекпотом.
- /simulators/roulette/lucky — Lucky Slots: бесплатные прокруты, категории наград, агрегированная статистика.
- /simulators/roulette/madness — Mutants Madness: уровневая система исследований, двойная валюта (жетоны + золото).
- /simulators/craft — крафт из четырёх категорий (Чёрная Дыра, Лаборатория, Орбы, Звёзды) с incentive-системой.

### Материалы
- /materials — пять вкладок: Сферы, Бустеры, Материалы, Здания, Зоны (обычные/люкс с расчётом вместимости хранения серебра).

### Справочники
- /bingo — все бинго-карты с табами, поддержка разбиения на части (32 мутанта → 2 доски по 16), динамические шапки по типу бинго.
- /tier-list — PvP тир-лист, данные из mut_tier.xlsx, мутанты расположены по генам.
- /top-evo — лидерборд игроков по эво, данные загружаются из Google Sheets (SSR).
- /credits — список разработчиков.

### API и бот
- /api/telegram-webhook — принимает файлы от Telegram-бота, парсит тиры, обновляет mutants.json через GitHub API.

## Ключевые механизмы

### Генный алгоритм скрещивания (breeding/)

Ядро — core-algorithm.ts. Для двух родителей с кодами генов (напр. AB и CD) генерируются все возможные комбинации потомков (AC, AD, BC, BD). engine.ts берёт настройки из settings.xml — гибридные веса (hybrid) определяют, как распределятся веса между однородными парами (AA, BB), разнородными (AB, AC...) и одиночными аллелями (A, B, C). Функция predict() возвращает отнормированный список Outcome с весами.

breeding.ts — оркестратор: сначала проверяет секретные рецепты (secretCombos.ts), потом считает генные комбинации, потом фильтрует по правилам типов (type-filters.ts — нельзя скрещивать GACHA/HEROIC/LEGEND без特定 условий), потом сортирует результаты. Обратный поиск (findParentsFor) перебирает пары родителей с ограничением в 5000 итераций.

### Крафт-симулятор (порт с Python)

craft-simulator.ts — TypeScript-порт craft.py. Рецепты хранятся в TXT-файлах с XML-подобной разметкой (<Recipe>, <ingredient>, <reward>). Парсинг через регулярные выражения, не через XML-парсер. Каждый рецепт имеет bonusPer1000 (шанс incentive-награды) и odds для каждой награды. Симуляция — цикл по N крафтам, на каждом шаге ролл по взвешенным odds + проверка incentive. Для предметов с регекс-ингредиентами (напр. "любая сфера ур.1") есть COMPLEX_REGEX_DESCRIPTIONS.

### Рулетки (cash/lucky/madness)

Все три следуют паттерну: контекст симуляции (pre-computed cumulative weights), кольцевой буфер истории (фиксированный размер, без роста памяти), пакетная обработка с поддержкой AbortSignal для отмены.

Cash Machine — базовая модель: рулетка по взвешенным наградам, подсчёт золота/серебра.

Lucky Slots — добавляет механику free-spin: когда награда помечена isFreeTry, прокрут добавляется обратно в очередь (queue += 1), что создаёт цепочки бесплатных вращений.

Mutants Madness — привязан к уровню игрока: каждый уровень открывает новые исследования (researchLevelMap), доступные награды фильтруются по максимальному исследованию. Двойная валюта: первые N прокруток за жетоны, остальные за золото.

### Поиск с нормализацией

search-normalize.ts реализует fuzzy-поиск для русских имён: транслитерация латинских букв в кириллические (B→В, H→Н, и т.д.), замена нуля на «о» (Ан0малия→аномалия), удаление кавычек, дефисов, скобок, спецсимволов. Это критично для базы мутантов, где имена содержат апострофы и латинские вкрапления.

### Текстуры и ассеты

mutant-textures.ts — канонизация ID мутанта (Specimen_BF_04 → Specimen_BF_04), поиск в JSON-маппингах (skin-textures.json, mutant-textures.json). bingo-textures.ts добавляет многоуровневые fallback-цепочки: скин-текстура → normal-текстура → ручная конструкция пути по папке. Все картинки в WebP с резервным JPG в BaseLayout.

### Telegram-интеграция

webhook-эндпоинт получает файл от бота, скачивает его через Telegram API, загружает текущий mutants.json из GitHub, парсит тиры из файла (tier-parser.js), валидирует значения и коммитит обновление обратно в репозиторий через GitHub Contents API.

## Что стоит отметить

- Нет тестов в src/ (только в node_modules) — вся валидация идёт через линтер и ручное тестирование.
- Python-скрипты в src/data/simulators/CRAFT/ — референсные реализации, не используются в продакшене.
- Git-история была очищена через filter-repo (см. .git/filter-repo/).
- В корне проекта лежат Lua-скрипты (*.lua.txt) — видимо, игровые скрипты/моды, не связанные с веб-приложением.
- Директория bot/ (Telegram-бот) отделена от основного веб-приложения.
- Версия проекта в package.json: 0.0.1 — проект не использует семантическое версионирование.
