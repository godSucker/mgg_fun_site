# character-textures — рендер full-body текстур мутантов

Автономный аналог «head»-парсера (`scripts/sync-mutants.ts`), но вместо готовых
thumbnail'ов он **сам собирает** полноразмерную стойку мутанта из атласа и геометрии.

Работает **без frida и без запущенной игры** — все источники плейнтекстовые и
без хэшей в пути, поэтому пайплайн пригоден для GitHub Actions.

## Источники (Kobojo CDN)

| Что | URL | Хэш нужен? |
|---|---|---|
| Геометрия | `gameconfig/sprites.xml` → `<Sprite id="specimen_<CODE>_stand" bitmap="character/<stem>.png">` | нет |
| Атлас | `assets/character/<stem>[_<tier\|skin>].png` (RGBA, lossless) | нет |
| Скины | `gameconfig/gacha.xml` → `<Gacha id="<skin>">` + `specimen="Specimen_XX_YY"` | нет |

> ⚠️ **`stem` берётся ТОЛЬКО из атрибута `bitmap`**, а не из кода. Они расходятся:
> site-код `cf_01` → атлас `c_f.png`, `aa_12` → `a_a_12.png`.

Мобильный маршрут (`assets/sprites/character/<code>-<HASH>.etc1.z` + `.etca.z`)
**намеренно не используется**: его хэши нигде в плейнтексте не публикуются, они
приходили из живой игры через frida — то есть ежедневная автономность невозможна.
Веб-PNG вдобавок lossless (лучше lossy ETC1) и существует даже для 6 мутантов,
у которых ETC1-атласа не было.

## Запуск

```bash
# Бэкфилл — разово, локально. Рендерит ВСЁ (~2567 изображений, ~1-2 часа).
python3 scripts/character-textures/sync_characters.py --full

# Инкремент — то, что крутится в sync-cron ежедневно.
# Опрос ~3100 атласов (16 потоков) ~2 мин, рендерит только новое/изменённое.
python3 scripts/character-textures/sync_characters.py

# Отладка
... --only aa_12            # один мутант (база + тиры + его скины)
... --limit 5               # остановиться после 5 рендеров
... --no-skins              # пропустить скины
... --sprites .cache/stands.xml   # взять геометрию из кэша (без фетча 222MB)
```

## Куда пишет

```
public/textures_by_mutant/<code>/FULL_<code>.png            # база
public/textures_by_mutant/<code>/FULL_<code>_<tier>.png     # bronze/silver/gold/platinum
public/textures_by_skins/textures_by_skin/full/FULL_<id>_<skin>.png
```

`rendered.json` — стейт-манифест `{путь: "Last-Modified|Content-Length|ETag"}`.
**Коммитится**, иначе инкремент теряется и CI перерисует всё заново.
`.cache/` — кэш (`stands.xml` ~21MB, скачанные атласы, `summary.md`), в git не идёт.

## Как это рендерит

`compositor.py` — копия проверенного `compositor_v11.py` из RE-воркспейса с **двумя**
правками под веб-маршрут (обе подтверждены совпадением с утверждённым рендером
пиксель-в-пиксель, 353×391):

1. **Y-flip отключён** — веб-PNG top-origin (мобильный декодированный атлас был перевёрнут).
2. **`qx=qy=1.0` при отсутствии `packSz`** — иначе на неквадратном атласе (869×585)
   fallback `pack=AW` даёт `qy=1.485` и мутанта растягивает по вертикали.

`frames.py` — выбор кадра стойки перенесён из `batch_render_stands.py` как есть
(`rest_frame` → `baked_frame` → эвристика). Кадр выводится из рига, референсы не нужны.

## Гарантии/ограничения

- **Слабые версии исключены** (`EXCLUDED_CODES = {a_02, b_02, c_02}` — Слабый Робот /
  Зомби / Воин). Они присутствуют в `sprites.xml` как `specimen_A_02_stand` и т.д.
  (делят атлас `a.png`/`b.png`/`c.png` с `_01`-версиями), но намеренно никогда не
  попадают в `mutants.json`, поэтому и текстуры им не нужны. Исключены везде,
  включая скины.

- **`POSE_OVERRIDES` (`ca_14`)** — брейкдансер, у него нет позы покоя; портрет
  отзеркален художником и из рига не выводится. Таблица намеренно размером 1 —
  **не расширять**, чтобы залатать кадр, который должно находить правило рига.
- **`segmented_rig()`** помечает риги «как `ca_14`» → в саммери попадает
  **NEEDS MANUAL REVIEW**. Рендер там — догадка; такое **не публиковать автоматически**.
- Приёмка — **глазами**. Метрика `similarity()` из RE-воркспейса за сессию пять раз
  двигалась против истины; здесь она сознательно не используется как гейт.
